Início CSS id e class Estilizando

CSS

O CSS significa Cascading Style Sheet, e é ele o principal para dar arrumação e estilização ao site.

O CSS têm três formas de ser adicionado ao código:

In line: Que se aplica o estilo na mesma linha/tag que você quer que seja estilizada
Interno: Que você adiciona dentro da head a tag astyle> e põe suas características desejadas nela
Externo: O mais comum e mais usado para sites grandes, que é criado um outro arquivo para adicionar o estilo e na head só linka o nome dele, geralmente feito por style.css

Separando com ID e Class

O ID deve ser único para cada elemento e não se pode repetir em seu código, já a classe, pode haver vários elementos com a mesma.

A class é chamada usando um ponto .
Já o id é chamado usando um sharp #




Padding, Margin e Border



A grosso modo a grande diferença é:



Margin / Margem - Espaçamento fora do conteúdo.

Padding / Preenchimento - Espaçamento interno. Dentro dos limites do conteúdo.


Tendo em conta que o limite do espaçamento do conteúdo de um elemento é a sua borda (border box). Portanto, a grande diferença neste caso será que a margem é aplicada já fora do elemento e criará espaçamento entre os restantes elementos, porém o padding irá criar espaçamento dentro do próprio conteúdo do elemento, não afetando o espaçamento dos outros elementos.



Ordem de prioridade(ESPECIFICIDADE)

Conteúdo encontrado no site da TipsCode

1. Id é maior que classe:

Se um elemento qualquer tiver um ID, e tiver 5 classes ao mesmo tempo. Ele irá satisfazer a ordem do ID


2. Mais específico

Se um elemento tiver 4 classes, e no CSS tiver uma alteração,
marcando 2/4 classes por exemplo, e outra marcando 3/4, a alteração será da mais específica no caso a 3/4.


3. CSS inline

Sempre que algo tiver o CSS in line, não tem nada nesse mundo que poderia alterar essa estilização. Então não é bom misturar CSS inline com CSS baseado em seletor.

4. !important

Sempre que tiver alguma alteração, mesmo que seja de baixa prioridade; ao colocar !important, ela terá prioridade máxima no navegador, nesse caso, tem como uma classe ser mais importante que um ID.
